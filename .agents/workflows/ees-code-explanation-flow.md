# 📖 EES 코드 분석 및 질문 워크플로우 (Code Explanation Flow)

이 워크플로우는 사원 평가 시스템(EES) 프로젝트의 코드를 분석하고, 이해가 가지 않는 부분에 대해 AI에게 효과적으로 질문하고 답변을 얻기 위한 절차를 정의합니다.

## 1. 질문 준비 단계 (Preparation)

질문을 할 때는 AI가 정확한 맥락을 파악할 수 있도록 다음 중 하나 이상의 방법을 사용합니다.

- **파일 참조**: `@파일명`을 사용하여 대화의 맥락에 해당 파일을 포함시킵니다.
- **라인 지정**: "35번 라인부터 50번 라인까지 로직이 궁금해"와 같이 구체적인 라인 번호를 언급합니다.
- **코드 스니펫**: 이해가 가지 않는 특정 코드 블록을 복사하여 질문에 포함합니다.

## 2. 질문 유형 선택 (Question Types)

더 나은 답변을 얻기 위해 원하는 설명의 깊이를 알려주세요.

- **전체 흐름 (Concept)**: "이 클래스가 하는 역할이 뭐야?" 또는 "이 기능의 전체적인 데이터 흐름을 알려줘."
- **세부 로직 (Detail)**: "이 if문 안에서 어떤 검증이 일어나는 거야?" 또는 "이 알고리즘의 동작 원리가 궁금해."
- **기술 스택 (Stack)**: "여기서 `@Transactional`은 왜 쓰는 거야?" 또는 "MyBatis 매퍼랑 어떻게 연결되는 거야?"
- **문제 해결 (Debug)**: "이 코드가 왜 이런 에러를 낼 수 있을까?"

## 3. 질문 가이드라인 (How to Ask)

질문을 명확하게 할수록 더 도움이 되는 답변을 받을 수 있습니다.

> **예시 1:** "@EmployeeServiceImpl.java 120라인에 있는 사원 권한 체크 로직이 이해가 안 돼. 한 줄씩 설명해 줄래?"
>
> **예시 2:** "MyBatis XML 매퍼에서 `select` 쿼리 결과를 어떻게 자바 Record 객체로 매핑하는지 흐름을 알려줘."
>
> **예시 3:** "우리 프로젝트에서 '비관적 락'을 적용한 부분이 어디고, 왜 그렇게 설계했는지 궁금해."

## 4. AI의 답변 방식 (Response Pattern)

AI는 사용자의 이해를 돕기 위해 다음과 같은 단계로 답변합니다.

1.  **개요 (Summary)**: 해당 코드의 핵심 목적을 쉬운 한국어로 요약합니다.
2.  **상세 분석 (Step-by-Step)**: 코드 블록 또는 라인별로 기능을 상세히 설명합니다.
3.  **시각화 (Optional)**: 복잡한 로직의 경우 Mermaid 다이어그램을 사용하여 흐름도를 그려줍니다.
4.  **관련 지식 (Tips)**: 해당 코드와 관련된 프로젝트 규칙(`.agents/code-style-guide.md`)이나 기술 정보를 추가로 제공합니다.
